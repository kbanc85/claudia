Showcase demo

This site doubles as a technical demonstration

The presentation is intentionally clean, but the underlying system is a content graph that converts writing into a reusable public knowledge base.

Update loop

  1. Capture A new article arrives through RSS, CMS webhook, or a publishing API connection.
  2. Process An API worker extracts summary, candidate claims, quotable lines, entities, and topics.
  3. Canonicalize Candidate claims are matched against the stable public claim library so the identity stays coherent.
  4. Publish The generator updates the HTML dossier plus claims.json, quotes.json, dossier.json, llms.txt, and feed.xml.
  5. Revalidate The site is rebuilt or revalidated so the public record and machine-readable endpoints stay in sync.

Machine-readable surfaces

dossier.json Full structured profile, proof set, claim library, work index, and recent writing. claims.json Canonical claims with support counts, article links, topics, and last-updated dates. quotes.json A clean quote bank with article lineage for press, researchers, and language models. llms.txt A plain-text briefing designed for AI systems that need the shortest correct orientation. feed.xml A standard feed surface for recent writing and downstream syndication.
